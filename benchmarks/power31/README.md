# POWER 3.1 Frozen Multilingual Benchmark (E1)

## Scope

This is a **synthetic benchmark** for POWER 3.1 retrieval evaluation.
It is **not human-annotated** and **not production evidence**.

### Dataset

- **228 queries** = 200 answerable + 20 no-answer + 8 distractor
- **100 corpus documents** (50 topics × 2 languages: UA + EN)
- **4 language-direction strata** (50 base answerable each):
    - `ua_to_ua`: Ukrainian query → Ukrainian document
    - `en_to_en`: English query → English document
    - `ua_to_en`: Ukrainian query → English document (cross-lingual)
    - `en_to_ua`: English query → Ukrainian document (cross-lingual)
- **Topic-driven design**: 50 frozen topics, each with explicit bilingual
  title, content, query, and atomic answer. Primary documents literally
  contain the atomic answer.
- **5 absent topics** (not in corpus) → 20 no-answer queries (1/stratum each)
- **8 QDD\* distractor queries** with primary + topically similar contradictory
  distractor document
- **Sparse qrels** (`qrels.synthetic.jsonl`): only positive primary/optional
  secondary and explicit distractor entries. Missing pair ⇒ relevance 0.
- **No-answer queries have no qrels entries** (all implicit zero).
- **Atomic answers** are real facts from the topic record, guaranteed to be
  substrings of the primary document content.
- **citation_document_ids** reference only primary relevant documents.

### Limitations

1. **Synthetic generation**: All data is generated by deterministic code
   (seed=42), not by human annotation.
2. **Topic-driven relevance**: Relevance is assigned by topic membership, not
   by human understanding.
3. **No real vault data**: The corpus is generated from templates and contains
   no private, real-world, or production notes.
4. **No production evidence**: Results from this benchmark must not be cited
   as evidence of production retrieval quality.
5. **CI gate only**: Suitable for hermetic CI regression testing, not for
   absolute quality claims.

## Usage

### Generate dataset

```bash
python benchmarks/power31/scripts/generate_benchmark.py
```

### Validate dataset

```bash
python benchmarks/power31/scripts/evaluation/validate_dataset.py
```

### Run regression tests

```bash
python -m pytest benchmarks/power31/tests/ -v --no-cov \
    --override-ini="addopts="
```

## Regeneration

The dataset can be deterministically regenerated at any time:

```bash
python benchmarks/power31/scripts/generate_benchmark.py
```

This will produce **bit-identical output** for the same seed (42).
To verify byte-identical regeneration, run the generator twice and
`diff` the output directory by relative path:

```bash
python benchmarks/power31/scripts/generate_benchmark.py
cp -a benchmarks/power31/dataset/v1 /tmp/v1_first
python benchmarks/power31/scripts/generate_benchmark.py
diff -r benchmarks/power31/dataset/v1 /tmp/v1_first
```
