# 📊 Звіт про тестування пам'яті ШІ-агентів у P.O.W.E.R. v2.0.3

Цей звіт містить детальний опис, архітектурний аналіз та результати запуску розширеного набору тестів для оцінки довготривалої та багатокрокової пам'яті ШІ-агентів у межах фреймворку **P.O.W.E.R. v2.0.3**. Тестування базується на концепціях та метриках чотирьох провідних світових стандартів оцінки пам'яті станом на **середину 2026 року**: **MemoryAgentBench (ICLR 2026)**, **LoCoMo**, **LongMemEval** та **BEAM**.

---

## 🔍 1. Концептуальний аналіз SOTA бенчмарків оцінки пам'яті агента

У 2026 році просте тестування LLM на розмір контекстного вікна (наприклад, Needle-in-a-Haystack) вважається застарілим. Агенти діють в інтерактивних середовищах, тому оцінка пам'яті переорієнтувалася на **поведінкові та крос-сесійні метрики**.

### 1.1 MemoryAgentBench (ICLR 2026)
* **Концепція:** *"Evaluating Memory in LLM Agents via Incremental Multi-Turn Interactions"*. Оцінює здатність агента накопичувати, фільтрувати та використовувати інформацію, яка надходить динамічно впродовж багатьох кроків взаємодії.
* **4 наріжні компетентності:**
  1. **Accurate Retrieval (AR) / Точний пошук:** Знаходження потрібного факту без додавання нерелевантного шуму.
  2. **Test-Time Learning (TTL) / Навчання на льоту:** Швидка адаптація та застосування щойно отриманої інформації на наступному кроці діалогу.
  3. **Long-Range Understanding (LRU) / Наскрізне розуміння:** Об'єднання взаємозв'язаних фактів, які розташовані у різних частинах історії або файлах.
  4. **Conflict Resolution (CR) / Вирішення конфліктів:** Успішне виявлення та заміна застарілих/суперечливих фактів новими (наприклад, оновлені IP-адреси чи токени).

### 1.2 LoCoMo (Long Context Memory)
* **Концепція:** Дослідження довготривалої пам'яті на проміжках понад 30+ сесій діалогу.
* **Ключові випробування:**
  * *Single-Hop Recall:* Прямий пошук факту з віддаленої сесії.
  * *Multi-Hop Reasoning:* Зв'язування фактів між сесіями (наприклад, сесія 1: "Мене звати А", сесія 12: "А працює в компанії Б", запит: "Де працює автор сесії 1?").
  * *Temporal Inference:* Розуміння хронологічної послідовності та пріоритету останньої події.

### 1.3 LongMemEval
* **Концепція:** Реалістичний стрес-тест на вилучення фактів із глибоких історичних логів.
* **Ключові метрики:**
  * *Lost-in-the-Middle:* Стійкість до втрати інформації, розташованої всередині великих масивів тексту.
  * *Abstention (Утримання від відповіді):* Здатність системи чітко констатувати відсутність інформації, замість генерації галюцинацій (Hallucinations).

### 1.4 BEAM (Behavioral Evaluation of Agent Memory)
* **Концепція:** Поведінковий аналіз згасання пріоритетів (Preference Decay) та оновлення правил/політик взаємодії користувача з агентом.
* **Фокус:** Виявлення застарілих інструкцій конфігурації та перевага нових політик над старими.

## 🔬 4. Аналітичні висновки та архітектурні орієнтири

Результати тестування дозволяють зробити кілька важливих архітектурних висновків щодо роботи пам'яті ШІ-агентів у P.O.W.E.R. v2.0.3:

1. **Захист від "Lost in the Middle":**
   Завдяки поєднанню **гібридного пошуку (RRF)** та **Cross-Encoder Reranker** (`ms-marco-MiniLM-L-6-v2`), фреймворк успішно витягує дрібні деталі з глибини довгих документів. Навіть коли потрібний факт оточений великим обсягом сміття, реранкер піднімає його релевантність у Top-1.
2. **Семантична нормалізація GraphRAG лінків:**
   Під час тестування виявлено, що YAML-валідність поля `related` вимагає жорсткої типізації. Впроваджений у версії v2.0.3 механізм авто-приведення (coerce) рядків у `TypedRelation` об'єкти дозволяє користувачу писати прості списки шляхів, не ламаючи Pydantic-схеми вбудованого GraphRAG.
3. **Ефективність виявлення конфліктів:**
   Використання косинусної близькості векторів `BAAI/bge-m3` у поєднанні з обмеженням мінімальної довжини тіла нотатки (50+ символів) виключає помилкові спрацьовування детектора ROT на коротких плейсхолдерах, але гарантує точне виявлення суперечностей у системних інструкціях та паролях.
